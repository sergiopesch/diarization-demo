# End-to-End Eval - 2026-04-27

## Scope

This eval verifies the browser app, Next.js API route, local-provider adapter,
and a deterministic mock local transcription worker. It is intended to catch
application wiring regressions before manual testing.

It does not verify real WhisperX model quality or real pyannote diarization
latency. Those require a running local worker with model dependencies,
downloaded models, and `PYANNOTE_AUTH_TOKEN`.

## Command

```bash
npm run test:e2e
```

`npm run verify` now includes the E2E suite after lint, unit tests, typecheck,
and build.

## Test Harness

- Runner: Playwright Chromium
- App server: Next dev server on an isolated E2E port
- Worker server: `tests/e2e/mock-local-worker.mjs`
- Mock worker behavior: accepts `/transcribe` and returns a fixed two-word
  diarized transcript
- Browser media: mocked in tests for deterministic live mic/system capture

## Covered Flows

| Flow | Coverage |
| --- | --- |
| Initial render | Verifies minimal live diarization UI, default backend, default model, and empty transcript state |
| WebM upload | Uploads a WebM file through the UI and verifies transcript rendering |
| Upload rejection | Uploads an unsupported MP3 file and verifies client-side rejection |
| Live mic capture | Mocks `getUserMedia` and `MediaRecorder`, captures a chunk, sends it through the API, and verifies transcript rendering |
| Live system audio capture | Mocks `getDisplayMedia`, captures a shared-audio chunk, sends it through the API, and verifies transcript rendering |

manual or automated pass with:
